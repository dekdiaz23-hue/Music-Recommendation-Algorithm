# Music Recommendation Algorithm
## Audio Clustering Project Unsupervised Machine Learning


## Project Overview

This project builds an unsupervised machine learning pipeline that
clusters songs by lyrical theme and generates recommendations. Using
a dataset of 28,362 songs spanning 1950 to 2019, we apply KMeans
clustering to group songs with similar lyrical DNA and recommend
songs from the same cluster to a target user.

No target variable exists. The algorithm discovers natural groupings
on its own using 11 lyrical theme scores per song.

## Notebook Descriptions

### 01_EDA.ipynb Exploratory Data Analysis

Performs univariate, bivariate, and multivariate analysis to
understand the dataset before modeling.

Key findings:
Four features had variance below 0.003 and were flagged for removal.
No feature pairs exceeded a correlation of 0.27 meaning all features
are independent and measure different lyrical content.
Hip hop scored dramatically higher on obscene than every other genre.
Violence and obscene have risen steadily since the 1980s while
world/life has declined over the same period.

Outputs: Observations and hypothesis for cleaning and modeling steps.


### 02_Cleaning_Preprocessing.ipynb Data Cleaning Current Version

Cleans and scales the raw training data based on EDA findings.

Steps performed:
1. Drops identity columns: Unnamed 0, lyrics, artist_name, track_name
2. Drops four low variance features: shake the audience, family/gospel,
   family/spiritual, dating
3. Confirms zero missing values
4. Removes outliers using IQR within each genre separately
5. Scales features using StandardScaler (mean=0, std=1)
6. Saves train_cleaned.csv, scaler.pkl, and cleaned_index.pkl

Note on versions: See the Two Versions section below for an
explanation of why outlier removal runs per genre in this version.


### 03_Modeling.ipynb Modeling Current Version

Trains a KMeans clustering model and generates recommendations.

Steps performed:
1. Loads cleaned data and aligns raw train.csv using cleaned_index.pkl
2. Finds optimal K using KneeLocator Elbow and Silhouette Score
3. Trains final KMeans model using k-means++ with n_init set to 20
4. Analyzes clusters using profile heatmap, genre breakdown, sample songs
5. Applies model to recommend.csv and generates recommendations
6. Validates model by checking hip hop cluster alignment

Results:
Optimal K = 10 because Silhouette Score was preferred over Elbow
Each of the 10 clusters maps cleanly onto one dominant lyrical theme
Cluster 7 which represents music about music showed the strongest
signal at 3.58


## Two Versions of Cleaning and Modeling

This repository contains two iterations of the cleaning and modeling
notebooks. The current versions listed above are the final versions
and should be used for all purposes.


### Version 1 Original

The original cleaning notebook applied IQR outlier removal globally
across all 28,362 songs at once. Because hip hop songs score
dramatically higher on obscene than every other genre, they appeared
as statistical outliers relative to the full dataset and were entirely
removed before clustering ran.

The original modeling notebook then tried to assign cluster labels
from the cleaned dataframe which had 17,172 rows directly to the raw
train.csv which had 28,362 rows, producing this error:

ValueError: Length of values (17172) does not match length of index (28362)

This was caused by the index mismatch between the cleaned and raw
dataframes after outlier removal eliminated a large portion of songs.


### Version 2 Current Fixed

Two changes were made to resolve both issues.

Fix 1 applied to Notebook 2:
Outlier removal now runs within each genre separately. A hip hop song
is only flagged as an outlier if it is extreme compared to other hip
hop songs, not compared to pop or country songs. This preserves hip
hop's legitimate signal in the dataset. Notebook 2 now also saves
cleaned_index.pkl which records exactly which row numbers survived
the cleaning process.

Fix 2 applied to Notebook 3:
The modeling notebook loads cleaned_index.pkl at the very top and
immediately creates an aligned version of train.csv called
df_raw_filtered using df_raw.loc[cleaned_index]. This ensures the
raw dataframe and the cleaned dataframe have identical row counts
before any cluster labels are assigned, permanently resolving the
length mismatch error.


## Dataset Columns

artist_name       Name of the artist
track_name        Name of the song
release_date      Year the song was released
genre             Categorical genre label
lyrics            Pre-tokenized lyrics (dropped before modeling)
len               Number of words in the lyrics
dating            Score 0 to 1 for dating related content
violence          Score 0 to 1 for violence related content
world/life        Score 0 to 1 for world and life themes
night/time        Score 0 to 1 for nightlife and time themes
shake the audience Score 0 to 1 for provocative content
family/gospel     Score 0 to 1 for family and gospel content
romantic          Score 0 to 1 for romantic content
communication     Score 0 to 1 for communication themes
obscene           Score 0 to 1 for obscene content
music             Score 0 to 1 for music about music
movement/places   Score 0 to 1 for movement and location themes
light/visual      Score 0 to 1 for visual and weather themes
family/spiritual  Score 0 to 1 for spiritual content
sadness           Score 0 to 1 for sadness themes
feelings          Score 0 to 1 for general emotional content
topic             Categorical lyrical topic label
age               Score 0 to 1 where 1 is oldest and 0 is newest
